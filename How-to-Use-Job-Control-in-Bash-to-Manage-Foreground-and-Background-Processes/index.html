How to Use Job Control in Bash to Manage Foreground and Background Processes

Posted on 2019-02-19 | Views:

In the situation when I use SSH to log on a remote computer or a server, I start a task in the foreground. The task is more bigger and complex, it will not be finished in a short peroid of time, but I need to close the terminal for some reasons. If I close the terminal, the process will be terminate too. I don’t want to stop it right now, how can I handle this situation?

Before you can deal with this case, you mush have some background knowledge about job control in Bash.

There 4 ways taht can terminate processes in our system.

  1. In a terminal, hold the control key, and press ‘c’, this will stop the process that we have started in the foreground.

  2. Passing signals to a process with the kill command.

  1. Closing the terminal
    Because the process is tied with the terminal instance that started it, when the terminal is closed, it will send a SIGUP signal to all of the processes tied to it.

Now I have a process running in the foreground. We need to keep it running when we close the therminal. Before we close the terminal, we will take sevelral steps to change it status, so when we come back, it will not be stopped by us.

  1. Ctrl-Z -> send a SIGSTP signal to the process, and push it back into the backgroud, then the process will be stopped.
  2. bg -> resume the process in the background again.
  3. You can type jobs to check the job and get its job number.
  4. disown %jobNUmber -> detach the job from the terminal
    The command disown will immune the SIGUP signal. Now we can close the terminal safely.

When we back, we can use the command top, ps aux, or pgrep -a name, to get the PID of the process that we detached before. But now we can only to kill it:(. So if you want to monitor the status of this process, you’s better to redirect its output into a file.
